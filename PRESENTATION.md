# 🧠 DataMind — презентация проекта

## Что это
Сервис, которому загружаешь CSV и задаёшь вопросы **обычными словами**
(«топ-3 категории по выручке»), а он сам пишет SQL, считает по базе и отвечает
по-человечески. Под капотом — цепочка LangGraph-агентов.

## Архитектура
```
Streamlit Cloud (UI, HTTPS)
        │  HTTP + X-API-Key
        ▼
FastAPI (Selectel 161.104.56.45:8000)   ←— агенты LangGraph + LLM
        │
        ▼
PostgreSQL (Docker, тот же сервер)        CSV-файлы → Selectel S3
```
- **Фронт** — тонкий клиент на Streamlit (только рисует, логика на бэке).
- **Бэк + БД** — в Docker на нашем сервере Selectel, работают 24/7.
- **Доступ к API** защищён ключом (`X-API-Key`).

## Стек
FastAPI · LangGraph · PostgreSQL · Selectel S3 (boto3) · Streamlit ·
LLM через polza.ai (`deepseek-v4-flash`) · uv · Docker.

## Как работает агент
```
router ──(ясно)──→ sql_agent ──→ synthesizer ──→ ответ
   │
   └──(неясно)──→ clarifier ──→ уточняющий вопрос
```
- **router** — решает: писать SQL или переспросить.
- **clarifier** — задаёт уточнение, помнит диалог (память по `session_id`).
- **sql_agent** — генерирует SQL по JSONB-данным, **сверяется с реальными значениями
  колонок** (не выдумывает «СПб» вместо «Санкт-Петербург»), сам исправляет ошибки.
- **synthesizer** — превращает результат в понятный ответ.

## Возможности
- **Загрузка CSV** → парсинг в БД + файл в S3, ссылка на скачивание (presigned, 15 мин).
- **Вопросы на естественном языке** с историей запросов.
- **Два агента**: «Агент 1» (наш, с памятью диалога) и «Агент 2» (с пошаговыми логами).
- **UI**: загрузка, лента датасетов, чат, история — пастельный стиль.

## Эндпоинты
`POST /upload` · `GET /datasets` · `GET /datasets/{id}/rows` ·
`GET /datasets/{id}/download` · `POST /query` · `POST /query2` · `GET /history`

## Деплой
UI — на Streamlit Cloud (из GitHub), бэкенд+БД — в Docker на Selectel.
Фронт и бэк живут раздельно и общаются по API — как в реальном проде.
